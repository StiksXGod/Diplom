# Дипломный проект

## Тема

**«Сравнительный анализ методов машинного обучения и обучения с подкреплением для прогнозирования и оптимизации торговых стратегий на биржевых котировках»**

## Цель проекта

Цель проекта состоит в разработке и исследовании программного комплекса на Python для сравнения нескольких подходов к принятию торговых решений на основе исторических биржевых данных. В рамках работы анализируется, насколько классические стратегии, методы машинного обучения и методы обучения с подкреплением способны прогнозировать направление движения цены и формировать более эффективные торговые стратегии.

## Научная и практическая актуальность

Научная актуальность проекта связана с ростом интереса к интеллектуальным системам анализа финансовых временных рядов. Финансовые рынки являются сложной, шумной и нестационарной средой, в которой особенно интересно сравнивать:

- простые эвристические стратегии;
- методы машинного обучения, работающие на заранее построенных признаках;
- методы обучения с подкреплением, способные обучаться через взаимодействие со средой.

Практическая актуальность заключается в возможности построить воспроизводимый исследовательский pipeline для оценки торговых подходов на реальных рыночных данных. Такой подход полезен для учебных исследований, анализа гипотез и демонстрации различий между методами принятия решений в задачах алгоритмической торговли.

## Какую проблему решает проект

Проект решает задачу единого и воспроизводимого сравнения разных подходов к торговле на биржевых котировках. На практике часто возникает проблема:

- разные стратегии оцениваются на разных данных;
- методы сравниваются по разным метрикам;
- результаты трудно воспроизвести;
- модели прогнозируют сигнал, но не всегда переводятся в понятную торговую логику.

В данном проекте все подходы сравниваются в рамках общей структуры:

- используется единый источник исторических данных;
- формируются одинаковые базовые признаки;
- данные делятся по времени без перемешивания;
- все стратегии приводятся к единому формату `portfolio_value`, `strategy_return`, `position`;
- результаты оцениваются единым набором финансовых метрик.

## Какие аналоги сравниваются

В проекте сравниваются следующие классы подходов:

### 1. Buy & Hold

Базовая стратегия, в которой актив покупается в начале периода и удерживается до конца. Используется как простой эталон для сравнения.

### 2. Moving Average Crossover

Правило на основе пересечения короткой и длинной скользящих средних. Если короткая средняя выше длинной, открывается длинная позиция, иначе позиция отсутствует.

### 3. ML-модели

Классические модели машинного обучения для прогнозирования направления движения цены:

- `LogisticRegression`
- `RandomForestClassifier`
- `GradientBoostingClassifier`

Их задача — предсказать бинарную цель `target_direction`, после чего предсказания переводятся в торговую стратегию.

### 4. RL-агенты

Агенты обучения с подкреплением, взаимодействующие с торговой средой `TradingEnv`:

- `DQN`
- `PPO`

Они принимают решения `hold / buy / sell` на основе окна исторических признаков и текущего состояния портфеля.

## Инструкция по запуску

### Полный эксперимент

Запуск полного pipeline:

```bash
python3 main.py \
  --ticker AAPL \
  --start_date 2018-01-01 \
  --end_date 2024-01-01 \
  --initial_cash 10000 \
  --transaction_cost 0.001 \
  --random_state 42
```

Что делает `main.py`:

1. Загружает котировки.
2. Генерирует признаки.
3. Делит данные на `train / val / test`.
4. Запускает benchmark-стратегии.
5. Обучает ML-модели.
6. Переводит ML-прогнозы в торговые стратегии.
7. Пытается подключить уже обученные RL-модели, если они существуют.
8. Считает финансовые метрики.
9. Сохраняет таблицы, графики и предсказания в `reports/`.

### Обучение PPO

```bash
python3 src/rl/train_ppo.py \
  --data_path data/processed/aapl_features.csv \
  --ticker AAPL \
  --timesteps 20000 \
  --initial_cash 10000 \
  --transaction_cost 0.001
```

### Обучение DQN

```bash
python3 src/rl/train_dqn.py \
  --data_path data/processed/aapl_features.csv \
  --ticker AAPL \
  --timesteps 15000 \
  --initial_cash 10000 \
  --transaction_cost 0.001
```

## Описание метрик

Для сравнения стратегий используются финансовые метрики:

- `total_return` — суммарная доходность стратегии за весь период;
- `annualized_return` — годовая доходность;
- `annualized_volatility` — годовая волатильность доходности;
- `sharpe_ratio` — отношение доходности к риску с учетом `risk_free_rate`;
- `max_drawdown` — максимальная просадка капитала;
- `win_rate` — доля прибыльных периодов;
- `number_of_trades` — число сделок или смен позиции;
- `calmar_ratio` — отношение годовой доходности к максимальной просадке.

Также для ML-моделей рассчитываются классификационные метрики:

- `accuracy`
- `precision`
- `recall`
- `f1`
- `roc_auc`

## Ограничения исследования

Проект имеет ряд ограничений, которые важно учитывать при интерпретации результатов:

- используется упрощенная торговая логика для одного актива;
- не моделируются сложные рыночные эффекты: проскальзывание, глубина стакана, частичное исполнение заявок;
- признаки строятся на основе исторических OHLCV-данных и не включают новости, макроэкономику и альтернативные данные;
- RL-среда является исследовательским приближением и не описывает реальную биржевую микроструктуру;
- параметры стратегий и моделей заданы как разумные базовые значения, но не претендуют на оптимальность;
- результаты зависят от выбранного актива, периода времени и рыночного режима;
- проект ориентирован на локальный запуск на ноутбуке, поэтому использует относительно легкие конфигурации моделей.

## Ожидаемые результаты

В результате выполнения проекта ожидается получить:

- воспроизводимый набор экспериментов по сравнению торговых подходов;
- таблицу метрик для rule-based, ML и RL-стратегий;
- графики доходности, сигналов и просадки;
- выводы о сильных и слабых сторонах каждого подхода;
- основу для аналитической части дипломной работы и дальнейшего расширения исследования.

Предполагается, что:

- `Buy & Hold` выступит в роли простого рыночного baseline;
- `Moving Average` покажет, насколько полезна простая техническая эвристика;
- ML-модели продемонстрируют способность извлекать сигнал из engineered features;
- RL-агенты позволят исследовать, может ли последовательное принятие решений улучшить результат по сравнению с фиксированными правилами.

## Результаты эксперимента

После запуска `main.py` результаты сохраняются в `reports/<ticker>_<start_date>_<end_date>/`, где можно найти:

- таблицы метрик;
- equity curves;
- графики стратегий;
- предсказания ML-моделей;
- действия и результаты RL-агентов, если они были доступны.
